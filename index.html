HOSTED CHECKOUT

Introducing Twinty Payment Solution's Hosted Checkout, a secure way to accept payments by redirecting users to a checkout link.

To start a transaction, provide customer information and make a server-side call to the /transaction/initialize endpoint. Upon success, receive a JSON response with an authorization URL and transaction ID.

Redirect the user to the URL for secure payment and seamless return to your website. Customize the process with metadata.

Twinty Payment Solution's Hosted Checkout simplifies payment and integrates seamlessly with your website for an improved checkout experience.

INVINCIBLE CHECKOUT

Introducing Invincible Checkout, a powerful integration offered by Twinty Payment Solution for seamless payment acceptance in three steps.

Step one: initiate a transaction with essential information and receive a JSON response with an authorization URL, reference, and transaction ID.
Step two: checkout process by sending a POST request with card information and reference code. Receive a response with transaction status, result code, and redirect URL.
Step three: check transaction status using requery feature with a GET request.
Invincible Checkout offers secure and hassle-free payment experience for customers and can be integrated into websites and applications. Handle failed transactions appropriately for a smooth user experience. Find more details in Twinty Payment Solution's documentation.
